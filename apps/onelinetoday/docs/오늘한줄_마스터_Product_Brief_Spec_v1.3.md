## 1. 제품 개요

### 1.1 기본 정보

| 항목 | 내용 |
|------|------|
| **앱 이름** | 오늘한줄 |
| **부제** | 하루 30초, 나를 다시 보다 |
| **한 문장 요약** | 매일 한 줄씩 써서 내 책 한 권을 만들어가는 마이크로 저널 앱 |
| **플랫폼** | Android (1차) · iOS (2차) |
| **기술 스택** | Flutter · SQLite (로컬) · RevenueCat · AdMob |
| **서버** | 없음 (100% 로컬 저장) |
| **스토어 카테고리** | 건강/운동 또는 라이프스타일 |

### 1.2 비전

사용자가 **자기 질책에서 벗어나 하루에 한 번 스스로를 칭찬하는 습관**을 잇도록 돕는 도구임. 하루의 한 줄이 모여 한 달 챕터가 되고, 한 해가 한 권의 책이 되는 **시간 누적형 자산 앱**임.

### 1.3 타겟 퍼소나

**1차 퍼소나 · "스스로 엄격한 30대"**
- 20후반~40대, 남녀
- 직장·육아·창업으로 바쁨
- 하루 성취를 과소평가하는 성향
- "오늘 뭐 했지" 자주 공허함
- 심리 상담까진 부담, 가벼운 셀프케어 도구 원함

**2차 퍼소나 · "감정일기 이탈자"**
- 20~30대, 여성 비중 높음
- 과거 감정일기 앱 사용 경험
- "쓸수록 지쳐서" 이탈
- 짧고 부담 없는 긍정 기록 도구 탐색 중

**3차 퍼소나 · "미니멀 노트 애호가"**
- 남녀, 성별·연령 폭 넓음
- Things 3·Bear·아침의달력 같은 단정한 도구 선호
- 과잉 기능 없는 깔끔한 앱 찾는 중

### 1.4 1인 운영 원칙

- 서버 0원 정책 (Firebase 무료 티어 이내 or 완전 로컬)
- API 호출 0원 정책 (무료 기능은 룰 기반)
- 콘텐츠는 템플릿 + 규칙 기반 (AI 호출 최소화)
- 출시 전 Red List 자가 검수 (법적 리스크 방어선)

## 2. 포지셔닝

### 2.1 포지셔닝 스테이트먼트

> **"오늘 잘한 거 한 줄, 내 책 한 페이지"** — 감정일기도 명상앱도 아닌, 하루에 30초만 투자해 자신을 칭찬하는 습관을 잇고 시간이 지날수록 나만의 책이 쌓이는 마이크로 저널

### 2.2 카테고리 프레이밍

| 구분 | 정의 |
|------|------|
| **내부 정의** | 마이크로 저널 · 셀프 칭찬 노트 |
| **외부 카테고리** | 라이프스타일 / 건강·운동 |
| **회피 카테고리** | 의료 · 심리상담 · 치료 |

### 2.3 차별화 4가지

1. **감정 기록이 아닌 "잘한 일" 기록** — 반추 유발 없음
2. **실패 경험 제거** — 프리셋 8개 + 기본 처리로 어떤 입력도 수용
3. **책 만들기 시스템** — 한 줄이 페이지가 되고 한 해가 권이 되는 시간 누적 자산 (v1.1 이전 스탬프 수집 시스템에서 전환됨)
4. **100% 로컬 저장 + 내보내기/불러오기** — 프라이버시 · 개인정보 비수집 · 기기 교체 시 데이터 보존

### 2.4 의료성 주장 회피 (요약)

**외부 카피 금지어** (Red List)
- 치료, 치유, 처방, 진단
- CBT, 인지행동치료, 심리치료, 심리상담
- 우울증, 불안장애, 공황장애, 번아웃 (질병명)
- 증상 완화, 효과 입증, 임상적
- 전문가 수준, 상담사 대체

**안전한 번역어** (Green List)
- 치료 → 돌봄 · 정리 · 응원
- 심리치료 → 셀프 케어 · 마음 돌봄
- 우울 → 지친 하루 · 힘든 날
- 증상 완화 → 하루 기분 정리 · 작은 변화
- 정신건강 개선 → 마음 돌봄 · 자기 인식

## 3. UX 핵심 원칙

본 앱의 모든 UX 결정은 아래 5가지 원칙에서 파생됨. 신규 기능이 원칙과 충돌할 경우 기능이 양보함.

### 원칙 1. 실패 경험 제거

어떤 입력도 "실패"로 분류되지 않음. 사용자가 `ㅋㅋ` 하나만 입력해도 유효한 페이지로 저장되고 응원 문구가 나옴. 빈 입력은 저장되지 않지만 오류 메시지도 띄우지 않음.

### 원칙 2. "잘한 일"의 재정의

"잘한 일" = 대단한 성취가 아니라 **하루를 살아낸 모든 증거**. 온보딩에서 이 정의를 명시적으로 학습시킴. "밥 챙겨 먹었다", "잠깐이라도 쉬었다", "힘들어도 버텼다" 모두 유효함.

### 원칙 3. 사용자 글 보존

앱은 사용자가 쓴 한 줄을 **절대 재작성하지 않음**. 리프레이밍은 챕터 자동 제목이라는 "옆에 붙는 레이어"로만 표현됨. "아무것도 못했다"라고 쓰면 그 글은 그대로 저장되고, 그 페이지는 그 달의 챕터에 포함됨.

### 원칙 4. 30초 원칙

핵심 사용 여정(앱 실행 → 입력 → 저장 완료)은 30초 이내 완료되어야 함. 이를 위해:
- 앱 실행 시 홈 화면이 기본 진입점 (온보딩 이후)
- 입력 필드가 자동 포커싱
- 저장 버튼이 항상 thumb zone
- 프리셋 칩 탭 → 즉시 입력 완료 가능

### 원칙 5. 감정 노출 최소화

감정을 분류·기록·통계화하는 UI 금지. 사용자는 "오늘 어떤 감정이었는지" 선택할 필요 없음. 감정 단어(힘들었다·기뻤다·우울했다)는 사용자가 자유 입력 시에만 등장하며, 앱이 먼저 제안하지 않음.

### 4.2 F1 · 한 줄 입력 상세

**목적**: 사용자가 하루에 한 번 "오늘 잘한 일"을 최대 200자로 기록함. 입력된 한 줄은 **새 페이지**로 취급됨.

**사양**:
- 최대 200자 (한글 기준)
- 최소 2자 이상 입력 시 저장 활성화
- 빈 입력 저장 시도 시 조용히 무시 (오류 메시지 없음)
- 하루 1건 제한 (같은 날 재입력 시 기존 기록 수정 or 새 기록 추가 선택)

**질문 (닉네임 반영)**:
- 기본: "오늘 잘한 일 하나, 적어볼까요?"
- 닉네임 설정 시: "○○님, 오늘 잘한 일 하나, 적어볼까요?"
- 변형 (3일마다 rotation, 로컬 상태): 30일 cycle
  - "오늘 뭐 하나라도 해냈나요?"
  - "오늘의 작은 한 줄은?"
  - "오늘, 기억해둘 만한 건?"
  - "스스로에게 칭찬하고 싶은 건?"
  - 등 총 10개 rotation

### 4.3 F2 · 페이지 추가 상세 (구 스탬프 부착)

**내부 로직 (백엔드)**: `오늘한줄_스탬프_템플릿_레퍼런스_v1.md` 3.1 참조. 키워드 매칭 엔진이 12종 내부 카테고리(stamp_id) 중 하나를 부여함. 이 값은 **챕터 자동 제목 생성** 및 **응원 문구 선택**에만 사용됨. UI에는 노출되지 않음.

```
입력 → 부분일치 키워드 스캔 → 12개 카테고리 중 첫 매칭 (내부 stamp_id)
  → 매칭 없으면 기본값 'recorded_day' 부착
  → 해당 카테고리의 3개 응원 문구 중 1개 선택
```

**UX 표현 (사용자 경험)**:
- 저장 직후 **페이지 추가 애니메이션**: 책이 한 페이지 두꺼워지고 페이지 카운터가 `+1` 올라감 (fade + scale 300ms)
- 화면 중앙에 "페이지 ○○" 숫자 크게 등장 후 축소
- 하단에 응원 문구 1줄 (4주 이내 중복 회피)
- 스탬프 이름·아이콘 **절대 UI 노출 금지**
- 5초 후 홈 화면으로 자동 전환 or 사용자 탭 시 즉시 전환
- v2.0 시점부터 우측 하단 "공유" 버튼 노출

### 4.4 F3 · 프리셋 헬퍼 상세

프리셋 헬퍼는 **2레이어 구조**임. 사용자가 느끼는 문턱에 따라 단계적 도움을 제공함.

- **Layer 1 — Sticky 칩 바**: 가벼운 선택지 · 즉시 삽입용 · 키보드 위 상시 노출
- **Layer 2 — 펼침 바텀시트**: 탐색형 옵션 · 카테고리별 대안 질문 · 문장 시작 어구

#### 4.4.1 Layer 1 — Sticky 칩 바

**트리거**: 입력창 포커스 진입 시 자동 노출 (키보드 등장과 동기화)

**위치**: 키보드 바로 위에 고정 (sticky). 키보드가 내려가면 칩 바도 함께 숨김.

**레이아웃**:
- **좌측 고정 영역** (세로 구분선으로 분리):
  - `뭔가 안 / 떠오를 때` 2줄 caption (text.micro)
  - 펼침 아이콘 `▾` (탭 시 Layer 2 오픈)
- **우측 가로 스크롤 영역**:
  - 칩 8개 가로 배열 · 우측 끝 페이드 그라디언트로 더 있음 암시

**칩 8개 (v1.3 확정 · v1.2와 동일)**:
1. 밥을 챙겨 먹었다
2. 잠깐이라도 쉬었다
3. 10분이라도 걸었다
4. 누군가와 대화했다
5. 새로운 걸 알게 됐다
6. 하루를 보냈다
7. 씻고 정리했다
8. 나를 챙겼다

**동작**:
- 칩 탭 시 해당 텍스트가 입력 필드에 즉시 삽입됨
- 선택된 칩은 `brand.default` 배경으로 전환되어 시각 피드백 제공
- 다른 칩 탭 시 이전 선택 칩은 기본 상태로 복귀 (단일 선택 모델)
- 사용자가 삽입된 텍스트를 추가 편집 가능 · 편집 없이 바로 저장해도 OK

**디자인 스펙**: `오늘한줄_디자인_토큰_v1.1.md` §7.5.1 참조

#### 4.4.2 Layer 2 — 펼침 바텀시트

**트리거**: Layer 1의 펼침 아이콘 `▾` 탭

**목적**: 칩 8개로 부족한 경우 **탐색형 옵션** 제공. 다른 관점의 질문이나 문장 시작 어구를 제시함.

**구성**:
1. **드래그 핸들** (상단) — 아래로 당겨 닫기 제스처
2. **헤더**: `오늘은 어떤 질문이 좋을까요?` + 닫기 × 버튼
3. **다른 질문 둘러보기** (4개 카테고리별 대안 질문):
   - `발견`: 오늘 발견한 작은 반짝임은?
   - `웃음`: 오늘 문득 웃게 한 순간은?
   - `대화`: 오늘 기억에 남는 한마디는?
   - `감각`: 오늘 기억나는 소리나 향기는?
4. **다른 질문 더 보기** 링크 — 탭 시 다른 4개 질문으로 교체 (내부 풀에서 회전)
5. **문장 시작하기 칩** (5개):
   - `"오늘은..."` · `"나는..."` · `"문득..."` · `"다행히..."` · `"그래도..."`
6. **CTA 버튼**: `이 질문으로 쓰기` (brand.default)

**동작**:
- 카테고리 질문 탭 시 해당 질문이 메인 화면 질문 영역을 교체함 (하루 3회까지만 교체 가능 — 신중한 선택 유도)
- 문장 시작 칩 탭 시 바텀시트 닫히고 입력창에 해당 어구 삽입 + 커서 위치 지정
- 배경 어두운 영역 탭 or 드래그 핸들 아래로 당김 시 취소 닫기

**데이터**:
- 칩 선택 시 `source='preset'` 기록 (v1.2 동일)
- 문장 시작 칩 선택 시 `source='starter'` 기록 (v1.3 신규)
- 카테고리 질문 교체 시 `question_id` 기록 (프리셋 카테고리 활용도 분석용)

**디자인 스펙**: `오늘한줄_디자인_토큰_v1.1.md` §7.8 참조

#### 4.4.3 스탬프 시스템 연동

프리셋 칩을 통한 입력은 **자유 입력과 완전히 동일하게 처리됨**. 즉:

- 칩 선택 → 입력창 삽입 → 저장 → F2 페이지 추가 플로우 진입
- F2 내부에서 키워드 매칭 엔진(`오늘한줄_스탬프_템플릿_레퍼런스_v1.md` §3.1)이 12종 내부 카테고리(stamp_id) 중 하나 부여
- 칩 문구는 키워드 매칭이 잘 되도록 설계되어 있음:
  - `잠깐이라도 쉬었다` → `rest_day`
  - `10분이라도 걸었다` → `moved_day`
  - `누군가와 대화했다` → `warm_day`
  - `새로운 걸 알게 됐다` → `learned_day`
  - `씻고 정리했다` → `cared_day`
  - `나를 챙겼다` → `cared_day`
  - `밥을 챙겨 먹었다` → `cared_day`
  - `잠깐이라도 쉬었다` → `rest_day`
  - `하루를 보냈다` → `recorded_day` (fallback)

`source='preset'` 플래그로 인해 향후 "프리셋 사용 비중 → 스탬프 분포 편향 여부" 분석이 가능함.

### 4.11 F13 · 데이터 내보내기 (v1.0 필수 편입)

**중요성**: 기기 교체·앱 재설치 시 데이터 유실은 앱 신뢰의 근본 문제. **v1.1 계획에서 v1.0으로 당겨 출시 필수 편입.**

**위치**: 설정(S10) → 데이터 관리 → "모든 기록 내보내기"

**출력 사양**:
- 파일 형식: JSON (`application/json`)
- 파일명: `오늘한줄_백업_YYYY-MM-DD.json` (예: `오늘한줄_백업_2026-04-23.json`)
- 인코딩: UTF-8
- 파일 구조:

```json
{
  "version": "1.0",
  "app": "OneLineToday",
  "exported_at": "2026-04-23T21:30:00+09:00",
  "device_id_hint": "abc123xyz",
  "entry_count": 127,
  "entries": [
    {
      "id": "uuid-v4",
      "content": "사용자 원문",
      "stamp_id": "achieved_day",
      "source": "direct",
      "created_at": "2026-04-23T09:15:00+09:00",
      "date": "2026-04-23",
      "updated_at": null
    }
  ],
  "preferences": {
    "nickname": "햇살",
    "reminder_enabled": true,
    "reminder_time": "21:30",
    "theme_mode": "system"
  }
}
```

**공유 방식**: 생성된 파일을 OS 공유 시트로 전달 → 사용자가 Google Drive · Dropbox · 카톡 나에게 보내기 · 이메일 등 선택

**자동 백업 권유**:
- 최초 백업까지 30일 이상 경과한 사용자에게 **비강제 1회 알림** 
- 설정 화면 상단에 "마지막 백업: YYYY-MM-DD" 표시 (망각 방지)
- 90일 이상 미백업 시 홈 화면 상단에 작은 안내 배너 (무시 가능)

### 4.12 F17 · 데이터 불러오기 (v1.0 신규)

**위치**: 설정(S10) → 데이터 관리 → "백업에서 복원"

**동작**:
1. 로컬 파일 선택기로 JSON 파일 선택
2. 버전·무결성 검증 (version 필드 확인, entries 배열 검증)
3. 기존 데이터 상태 확인:
   - 기존 데이터 없음: 바로 가져오기
   - 기존 데이터 있음: 선택 다이얼로그 노출
     - **병합**: 중복 id는 무시, 새 id만 추가
     - **덮어쓰기**: 기존 전부 삭제 후 백업 파일로 대체 (확인 다이얼로그 필수)
     - **취소**: 아무 변경 없음
4. 완료 시 성공 토스트: "N개 기록을 가져왔어요"
5. 실패 시 명확한 오류 메시지 (파일 손상·버전 미지원 등)

**버전 호환성 정책**:
- v1.0 백업 파일은 향후 모든 버전에서 읽기 가능하도록 하위 호환 보장
- 향후 스키마 추가 시 누락 필드는 기본값으로 채움

**데이터 저장**: SharedPreferences의 `nickname` 키. 로컬만 저장, 외부 전송 없음.

### 4.14 F19 · 오늘의 한마디 (v1.0 신규)

**목적**: 홈 화면 상단에 **소확행 감성의 짧은 문장**을 노출하여, 사용자가 하루의 소소한 감각에 주목하도록 환기함. 명상 앱의 "오늘의 명언"과 달리 **일상의 작은 발견을 유도하는 부드러운 제안** 형식임.

**위치**: 홈 화면 (S3) 상단 · 챕터 헤더 바로 아래 · 날짜/질문 영역 위

**컨텐츠 성격**:
- 20자 내외의 짧은 제안형 문장
- 명령형 X (`~해보세요` 금지) · 제안형 O (`~할까요?` 권장)
- 추상적 개념 X (`행복`, `성장` 금지) · 구체적 감각 O (`창밖 바람`, `따뜻한 물` 권장)
- 톤: 부드럽고 따뜻한 존댓말

**예시**:
- `오늘은 하늘 한 번 올려다볼까요?`
- `창문을 여니 바람이 따뜻해졌어요`
- `따뜻한 물 한 잔도 충분히 위로가 돼요`
- `출근길 어떤 색이 제일 많이 보였나요?`
- `아침에 눈 뜬 것만으로도 오늘의 몫은 했어요`

**데이터 소스**: 로컬 SQLite `daily_quotes` 테이블 (v1.0 기준 100개 프리로드)

**노출 로직**:
- 오늘 날짜 기반 seed로 결정적 선택 (동일 날짜에 앱 재진입 시 동일 한마디 유지)
- 28일 이내 중복 회피
- 100% 로컬 · **서버·LLM 호출 없음** (1인 운영 원칙 준수)

**UI 스펙**: `오늘한줄_디자인_토큰_v1.1.md` §7.9 참조

**v2.0 고려사항**:
- 사용자 투고 기능 (큐레이션 후 반영)
- 계절·시간대별 가중치 노출 (아침/저녁 구분)
- Firebase Remote Config로 A/B 테스트 (내부 템플릿 성능 관측)

### 4.15 F20 · 어제의 페이지 카드 (v1.0 신규)

**목적**: 사용자가 **어제 작성한 한 줄**을 입력창 하단에 부드럽게 회상 노출하여, 작성 연속성 감각과 아카이브 진입 동기를 동시에 제공함. 책 메타포 일관성 유지를 위해 `어제의 한 줄` 대신 **`어제의 페이지`** 로 네이밍함.

**위치**: 홈 화면 (S3) 입력창 하단 · 메인 CTA 버튼 위

**표시 내용**:
- 라벨: `어제의 페이지`
- 본문: 어제 작성한 한 줄 (이탤릭 처리)
- 좌측 3px `brand.default` 보더 (시각 포인트)

**노출 조건**:
- 어제(= 현재 날짜 -1일) 작성한 페이지가 있을 때만 표시
- 어제 미작성 시: 카드 비노출 (화면 여백으로 대체 · 죄책감 유발 회피)

## 7. 데이터 모델

### 7.1 저장 정책 (메타포 중립 원칙 명시)

- **저장소**: SQLite (엔트리) + SharedPreferences (설정)
- **동기화**: 없음 (로컬 전용)
- **백업/복원**: F13/F17 로컬 JSON 내보내기·불러오기 (v1.0 포함)
- **개인정보**: 서버 전송 없음, 앱 제거 시 자동 소실
- **user_id**: 현재 스키마에 미포함 (섹션 11.6의 도입 트리거 충족 시 마이그레이션으로 추가)
- **메타포 중립 원칙**: 데이터 테이블·컬럼·내부 필드명은 "책·페이지·챕터·스탬프" 등 표면 메타포 용어 금지. 사용자에게 보이는 메타포가 바뀌어도 데이터는 그대로 유지되도록 설계함.

### 7.2 Entry 테이블 (SQLite) — 변경 없음

```sql
CREATE TABLE entries (
  id TEXT PRIMARY KEY,              -- UUID v4
  content TEXT NOT NULL,            -- 사용자 입력 원문 (max 200자)
  stamp_id TEXT NOT NULL,           -- 내부 카테고리 ID (챕터 자동 제목용) · UI 미노출
  source TEXT NOT NULL,             -- 'direct' | 'preset'
  created_at TEXT NOT NULL,         -- ISO8601 DATETIME
  date TEXT NOT NULL,               -- YYYY-MM-DD (groupby용)
  updated_at TEXT                   -- 수정 시 갱신 (nullable)
);

CREATE INDEX idx_entries_date ON entries(date);
CREATE INDEX idx_entries_stamp ON entries(stamp_id);
```

> **중요**: `stamp_id` 필드명은 v1.2 책 메타포 채택 이후에도 **그대로 유지함**. 이유는 메타포 중립 원칙 — "책 모드에서는 내부 카테고리로만 쓰이다가, 만약 개발 중 스탬프 모드로 되돌아가기로 결정되면 이 필드를 UI에 그대로 재노출할 수 있음". 테이블·컬럼 마이그레이션 없이 UI 레이어만 교체 가능.
>
> 페이지 번호는 별도 컬럼 없이 `ORDER BY created_at` 쿼리로 런타임 계산함. 챕터는 `strftime('%Y-%m', created_at)` 그룹핑. 권(Volume)은 `strftime('%Y', created_at)` 그룹핑.

## 8. 디자인 방향

### 8.1 선택된 방향

> **"따뜻한 종이 위의 짙은 라벤더 잉크" + "내가 쓰는 책"**

v1.1에서 채택한 5색 팔레트 기반 디자인은 그대로 유지함. v1.2에서는 이 디자인 언어 위에 **"책을 쓰고 있다"는 시각 메타포**를 얹음. 배경은 종이의 피치 크림, 브랜드 색은 책 제본의 짙은 라벤더, 책 표지·페이지·서재 구조는 모두 이 두 색을 기본으로 디자인됨.

### 8.2 무드 키워드

- 종이의 (paper-like)
- 모던한 (modern, not retro)
- 따뜻한 (warm)
- 겸손한 (humble, not loud)
- 저녁의 (evening, reflective)
- **책의 (book-like) — v1.2 추가**

### 8.3 레퍼런스 앱 · 시각 참고

- **Things 3** — 여백·플랫·정갈함
- **Bear** — 종이 위 글쓰기 톤
- **아침의달력** — 한국적 감성의 부드러움
- **Apple Books** — 디지털 서재 표현의 정갈함 (v1.2 추가)

### 8.8 컴포넌트 스타일 원칙 (책 관련 추가)

**책 표지 (Book Cover)**
- 크기: 세로 직사각형 (4:5 비율)
- radius 6 · border 0.5px border-default · shadow 없음 (플랫)
- bg: brand (진행 중인 책) / accent 색상 순환 (완성된 권마다 다른 색)
- 텍스트: 연도 Display 크기 · 세로 중앙 정렬 · text on brand

**페이지 번호 인디케이터**
- 크기: Display (24px) 또는 Title (18px)
- 색: brand
- 표시: "127p" · "1권 245p"

**서재 그리드**
- 책들 가로 배열 (flex-start)
- 각 책 width 60~80px
- 책 사이 간격 4px (붙어 있는 느낌)

**챕터 구분선**
- 캘린더·아카이브에서 월이 바뀔 때 얇은 0.5px 선 + 챕터 제목 작게

### 11.4 v1 코드 구조 준비 — 메타포 전환 가능성 보존 3원칙 (개발 중 롤백 대비)

> **중요**: 개발 단계에서 주변 반응을 보며 책 → 스탬프로 전환할 가능성을 열어둠. 아래 3원칙을 지키면 전환 공수가 UI·콘텐츠 재작성 수준으로 한정됨 (데이터·로직은 그대로 유지).

**원칙 1. 데이터 테이블·컬럼명을 메타포 중립으로 유지**
- 테이블: `entries` (O) / `book_pages`·`chapters` (X)
- 컬럼: `stamp_id` 그대로 유지 (내부 카테고리 의미로 재해석)
- 페이지 번호·챕터·볼륨은 **런타임 쿼리로 계산** (전용 컬럼 생성 금지)

**원칙 2. UI 컴포넌트를 메타포 레이어와 분리**
- `EntryCard` · `CategoryBadge` · `CelebrationView` — 메타포 없는 이름
- "페이지"·"챕터" 같은 메타포 용어는 컴포넌트 prop 또는 별도 라벨 서비스에서 주입
- 전환 시 라벨·프레젠터 레이어만 교체

**원칙 3. 챕터·볼륨 자동 제목 로직을 별도 서비스로**
- `ChapterTitleService` · `VolumeSubtitleService`로 독립
- 메타포 전환 시에도 재활용 가능 (주간 리포트·회고 등에서 그대로 사용)
- 단위 테스트 확보로 안전성 보장

**추가 v2.0 대비 준비 (v1.1에서 계승)**
- Repository 패턴 적용 (`EntryRepository`)
- `EntryCard` 위젯 재사용 가능 설계 (v2.0에서 RepaintBoundary로 감싸 이미지 출력)
- RevenueCat 초기화 시 익명 ID 로깅 (`revenuecat_anon_id`)

### 12.1 용어 정리

| 용어 | 정의 |
|------|------|
| 엔트리 (Entry) | 사용자가 작성한 하루 한 줄 기록 단위 · 1 페이지와 동일 |
| **페이지 (Page)** | **엔트리의 사용자 대면 명칭. 매일 한 페이지씩 추가됨** |

| **볼륨 · 권 (Volume)** | **한 해 분량의 챕터 모음. 연 종료 시 자동 부제 부여** |
| **서재 (Library)** | **완성된 볼륨들이 진열된 뷰** |
| 내부 카테고리 (Internal Category) | 키워드 매칭으로 부여되는 12종 라벨. v1.2부터 UI 미노출. 구 "스탬프" |
| 프리셋 (Preset) | 입력 화면의 8개 예시 칩 |
| 주간 리포트 | 매주 일요일 생성되는 템플릿 기반 요약 |
| 공유 카드 (Share Card) | v2.0 공유용 이미지 (1080×1080 또는 1080×1350) |
| Red List | 의료성 회피를 위한 외부 카피 금지어 목록 |
| Green List | Red List의 안전한 대체 표현 목록 |
| Tier 1 유료 | RevenueCat 익명 ID로 처리 가능한 로컬 유료 기능 |
| Tier 2 유료 | 서버 호출이 필요한 고도화 유료 기능 (user_id 필요) |
| **닉네임 (Nickname)** | **앱이 사용자를 부르는 선택 입력 이름. F18** |
| **백업 (Backup)** | **F13으로 내보낸 JSON 파일. F17로 복원 가능** |
